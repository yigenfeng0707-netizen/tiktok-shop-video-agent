# iBot 杭州站黑客松 - 技术路线

## 一、使用的模型

### 大语言模型（LLM）

| 模型 | 用途 | 选型理由 |
|------|------|---------|
| GPT-4 | 脚本生成、数据分析 | 多语种能力强、创意写作能力突出 |
| Claude | 脚本生成、文化禁忌判断 | 长上下文理解、安全合规能力强 |

### 数字人 API

| 服务商 | 用途 | 选型理由 |
|--------|------|---------|
| HeyGen | 数字人视频合成 | 多语种 TTS、面孔多样性、API 成熟 |
| 腾讯智影 | 数字人视频合成（备选） | 国内访问稳定、中文优化、成本低 |

### TTS 多语种

- 支持语言：英语、越南语、阿拉伯语、印尼语
- 服务集成：HeyGen 内置 TTS / 腾讯智影 TTS
- 特色：支持语速调节、情感标注、口音选择

## 二、工具平台

### 主编排平台

- **MiXer AI**：主编排平台，负责 4 个子 Agent 的调度、状态管理、数据流转
- 核心能力：可视化工作流、Agent 编排、变量传递、错误重试

### MCP 协议

- **open.hirebox.cn**：通过 MCP 协议暴露 Agent 能力
- 暴露能力：
  - 脚本生成接口
  - 视频合成接口
  - 批量剪辑接口
  - 数据复盘接口

### 视频处理

- **ffmpeg**：批量剪辑、字幕烧录、BGM 混音、格式转换
- 核心命令：视频拼接、字幕硬编码、音频混合、分辨率统一

## 五、工作流：4 步流水线架构

### 架构总览

```
[用户输入] → [MiXer AI 主编排] → [4 个子 Agent 流水线] → [多国视频产出]
                                    ↓
                    ┌───────────────┼───────────────┐
                    ↓               ↓               ↓
              [脚本生成Agent]  [数字人Agent]  [批量剪辑Agent]
                                                    ↓
                                            [数据复盘Agent]
```

### 步骤 1：脚本生成子 Agent

- **输入**：产品标题、卖点、目标国家
- **处理**：
  - 调用 GPT-4/Claude 生成多国本地化脚本
  - 文化禁忌规则引擎校验
  - 热点元素融入（节日、挑战赛、流行梗）
- **输出**：5 国本地化脚本（JSON + Markdown）

### 步骤 2：数字人子 Agent

- **输入**：本地化脚本、目标国家
- **处理**：
  - 调用 HeyGen/腾讯智影 API
  - 多语种 TTS 配音生成
  - 数字人面孔匹配（按目标国人群特征）
- **输出**：5 国数字人讲解视频片段

### 步骤 3：批量剪辑子 Agent

- **输入**：数字人视频片段、字幕、BGM、产品画面
- **处理**：
  - 调用 ffmpeg 批量合成
  - 字幕硬编码
  - BGM 混音
  - 产品画面插入
  - 统一分辨率（1080×1920）
- **输出**：5 国本地化短视频成片（mp4）

### 步骤 4：数据复盘子 Agent

- **输入**：视频发布后的数据（播放量、完播率、转化率）
- **处理**：
  - 调用 GPT-4 分析数据
  - 识别高效脚本模式
  - 输出优化建议
- **输出**：数据复盘报告、脚本优化建议
- **闭环**：优化建议反哺脚本生成子 Agent

## 六、MCP 协议说明

### 协议实现

- 通过 **open.hirebox.cn** 生成 MCP 协议接口
- 将 4 个子 Agent 能力标准化暴露

### 暴露的能力接口

| 接口名称 | 功能 | 输入 | 输出 |
|---------|------|------|------|
| generate_script | 脚本生成 | 产品信息、目标国家 | 多国脚本 |
| synthesize_video | 数字人合成 | 脚本、目标国家 | 数字人视频 |
| batch_edit | 批量剪辑 | 视频片段、字幕、BGM | 成片 mp4 |
| review_data | 数据复盘 | 发布数据 | 优化建议 |

### 协议优势

- 标准化接口，便于第三方集成
- 支持 MiXer AI 主编排调度
- 解耦子 Agent 实现，便于独立迭代
- 支持未来扩展更多子 Agent
