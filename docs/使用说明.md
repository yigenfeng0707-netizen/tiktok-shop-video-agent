# iBot 杭州站黑客松 - 使用说明

## 一、输入说明

### 必填输入项

| 输入字段 | 类型 | 说明 | 示例 |
|---------|------|------|------|
| 产品标题 | 文本 | TikTok Shop 商品标题 | 便携榨汁杯 USB 充电式 |
| 卖点列表 | 数组 | 核心卖点 3-8 条 | ["USB充电","30秒出汁","便携折叠","食品级材质"] |
| 目标国家 | 多选 | 美国/越南/沙特/英国/印尼 | ["美国","越南","沙特","英国","印尼"] |
| 平台风格 | 单选 | TikTok 短视频 / 直播切片 | TikTok 短视频 |

### 选填输入项

| 输入字段 | 类型 | 说明 | 示例 |
|---------|------|------|------|
| 实拍素材链接 | URL | 已有产品实拍视频/图片链接 | https://... |
| 品牌调性 | 文本 | 品牌/店铺风格描述 | 年轻活力、性价比 |
| 价格区间 | 文本 | 商品售价区间 | $15-25 |
| 目标人群 | 文本 | 目标消费者画像 | 18-35 岁都市女性 |

## 二、输出说明

### 输出物清单

1. **多国本地化短视频成片（mp4）**
   - 格式：MP4 (H.264)
   - 分辨率：1080×1920（竖屏）
   - 时长：30-60 秒
   - 数量：按选择的目标国家数量输出（1-5 条）
   - 包含：数字人出镜 + 字幕 + BGM + 产品画面

2. **脚本文件（json + md）**
   - 格式：JSON 结构化数据 + Markdown 可读版本
   - 内容：分镜脚本、台词、镜头描述、时长标注
   - 多语言版本：每个目标国一份本地化脚本

3. **字幕文件（srt）**
   - 格式：SRT 标准字幕格式
   - 语言：对应目标国语言（英语/越南语/阿拉伯语/英语/印尼语）
   - 时间轴：与视频成片对齐

## 三、使用步骤

### 5 步操作流程

#### 步骤 1：输入产品信息
- 填写产品标题
- 列出核心卖点（3-8 条）
- 可选：上传实拍素材、填写品牌调性

#### 步骤 2：选择目标国家
- 多选目标国家（美国/越南/沙特/英国/印尼）
- 选择平台风格（TikTok 短视频 / 直播切片）
- 可选：填写目标人群、价格区间

#### 步骤 3：生成脚本
- 脚本生成子 Agent 自动运行
- 输出 5 国本地化脚本（含文化适配、热点融入）
- 文化禁忌规则引擎自动校验
- 用户可预览/微调脚本

#### 步骤 4：数字人合成
- 数字人子 Agent 调用 HeyGen/腾讯智影 API
- 按脚本生成多语种数字人讲解视频
- 自动匹配目标国面孔特征
- 自动生成多语种 TTS 配音

#### 步骤 5：批量剪辑输出
- 批量剪辑子 Agent 调用 ffmpeg
- 合成数字人 + 字幕 + BGM + 产品画面
- 输出 5 国本地化短视频成片（mp4）
- 同步输出脚本文件、字幕文件

## 四、适用边界

### 支持的目标国家（5 国）

| 国家 | 语言 | 文化注意事项 |
|------|------|-------------|
| 美国 | 英语 | 注意种族敏感词、宗教中立 |
| 越南 | 越南语 | 注意春节、中秋等节日营销节点 |
| 沙特 | 阿拉伯语 | 严格遵守伊斯兰文化禁忌、斋月特别规则 |
| 英国 | 英语 | 注意英式英语表达、避免美式俚语 |
| 印尼 | 印尼语 | 注意穆斯林人口占多数、清真相关规则 |

### 支持的平台

- **TikTok 短视频**：30-60 秒竖屏视频，符合 TikTok Shop 种草视频规范
- **直播切片**：30-60 秒竖屏视频，从直播内容提取精华片段

### 视频时长

- 标准时长：**30-60 秒**
- 最优完播率时长：45 秒
- 不支持：60 秒以上长视频（超出 TikTok 短视频最佳实践）

### 不支持的场景

- 横屏视频（非 TikTok 平台）
- 60 秒以上长视频
- 非上述 5 国的其他国家
- 非电商类内容（如纯娱乐、新闻等）

## 七、MCP 连通性测试

### 部署后验证

项目部署到 MiXer AI 后，需验证 MCP 链接是否正常工作。运行连通性测试脚本：

```bash
# 不带鉴权 token
python demo/test_mcp_connectivity.py <mcp_url>

# 带 API Key 鉴权
python demo/test_mcp_connectivity.py <mcp_url> --token <api_key>
```

也可以双击 `demo/test_mcp_connectivity.bat`，按提示输入 MCP 链接和 API Key 运行。

脚本仅依赖 Python 标准库（urllib.request、json、time、sys），无需安装任何第三方包。

### 测试项目

脚本依次执行以下 5 项测试并输出格式化报告：

| 序号 | 测试项 | 说明 |
|------|--------|------|
| 1 | 连通性测试 | 发送 `tools/list` JSON-RPC 请求，检查是否返回工具列表 |
| 2 | 响应时间测试 | 测量 `tools/list` 往返时间，判断是否 < 3 秒 |
| 3 | 工具列表统计 | 解析工具列表，打印每个工具的 name 和 description |
| 4 | 工具调用测试 | 对每个工具发送 `tools/call` 调用（空参数），检查是否返回有效响应 |
| 5 | 稳定性测试 | 连续 3 次请求 `tools/list`，检查响应一致性 |

### 评分标准（100 分制）

| 评分项 | 满分 | 评分规则 |
|--------|------|----------|
| 连通性 | 40 分 | 成功返回工具列表得 40 分；连通但无工具得 20 分；失败得 0 分 |
| 响应时间 | 30 分 | 往返时间 < 3 秒得 30 分；3-5 秒得 15 分；> 5 秒得 5 分；失败得 0 分 |
| 工具调用 | 30 分 | 按成功调用的工具比例计分：`30 × 成功工具数 / 总工具数` |

### 等级划分

| 等级 | 总分区间 | 含义 |
|------|----------|------|
| A | ≥ 90 | 优秀，MCP 链接完全正常 |
| B | 80 - 89 | 良好，基本可用 |
| C | 70 - 79 | 一般，存在轻微问题 |
| D | 60 - 69 | 及格，需要排查问题 |
| E | < 60 | 不及格，MCP 链接异常 |

### 常见错误排查

| 错误现象 | 可能原因 | 排查方法 |
|----------|----------|----------|
| 网络错误 / 连接被拒绝 | MCP 服务未启动或地址错误 | 确认 MiXer AI 部署状态，检查 URL 是否正确（应为完整 MCP 端点地址） |
| 超时（10 秒无响应） | 服务冷启动或网络延迟过高 | 等待服务预热后重试；检查 MiXer AI 平台运行日志 |
| HTTP 401 / 403 | 鉴权失败，API Key 缺失或无效 | 使用 `--token` 传入正确的 API Key；确认可见范围与权限配置 |
| HTTP 404 | 端点路径错误 | 确认 MCP 端点 URL 路径（通常以 `/mcp` 结尾） |
| HTTP 500 | MCP 服务端内部错误 | 查看 MiXer AI 应用日志与 Trace；检查工具实现是否有异常 |
| JSON 解析失败 | 返回内容非 JSON-RPC 格式 | 确认端点为标准 MCP 协议（JSON-RPC 2.0 over HTTP） |
| 工具列表为空 | 工具未正确注册 | 检查 MCP 服务端工具注册逻辑 |
| tools/call 全部 FAIL | 工具实现异常或参数校验过严 | 查看具体错误信息；工具调用测试仅校验不超时、不返回 500 |
| 稳定性测试不一致 | 服务存在间歇性故障 | 多次重试；检查服务负载与资源占用 |

> 提示：脚本退出码为 0 表示总分 ≥ 90（达到 A 级），退出码为 2 表示未达标，便于在 CI/CD 或批处理中自动判断。

## 八、定价方案

| 模式 | 价格 | 适用场景 |
|------|------|---------|
| 按条计费 | 9.9 元/条 | 偶尔需要、试水用户 |
| SaaS 订阅 | 999 元/月 | 中小卖家日常使用（含 200 条/月） |
| 企业定制 | 3-10 万元/项目 | 大卖家私有化部署 |

### 说明

- **按条计费**：无需订阅，按实际生成视频条数计费，适合偶尔有视频需求的试水用户
- **SaaS 订阅**：月付 999 元，每月含 200 条视频生成额度，超出部分按 9.9 元/条计费，适合中小卖家日常运营
- **企业定制**：按项目制收费，提供私有化部署、多国本地化、定制品类、数据看板等企业级服务，适合中大型跨境卖家和品牌方
